Detailed Description

Various definitions for the CANopen implementation.

This file defines a basic number of objects and messages for the CANopen implementation provided by mbed-lib, and it provides function definitions.

)

Send a CAN message via CANopen protocol.

This routine sends a message via the CAN interface. If the message could not be sent (because the internal transmit buffers are full), the message is buffered and sent when when of the internal transmit buffers becomes availble.

Parameters:
portNoCAN interface to send on. Should be MBED_CAN0 or MBED_CAN1.
nodeIDID of the CAN device the message should be sent to.
cobTypeCOB type of the message.
lenMessage length.
dataData to be sent. The total number of bytes that will be sent depends on the provided message length, but will never be more than eight bytes.
Returns:
0 if a message was successfully sent or buffered, or an integer larger than 0 when both sending and buffering failed.
